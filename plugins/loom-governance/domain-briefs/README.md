# Domain-brief registry

Consolidated worker briefs for technical domains (replaces the former 7 `sdd-domain-*`
plugins, collapsed into the governance core in v3.1.0). Consumed via
`get_domain_brief <domain>` in `.logic-loom/scripts/bash/common.sh`, keyed from
`domains.conf`. One `<domain>.md` per domain: frontend, backend, database, testing,
security, performance, devops.
