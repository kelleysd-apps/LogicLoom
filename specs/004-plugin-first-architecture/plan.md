# Implementation Plan: Plugin-First Architecture (v4.0)

**Branch**: `004-plugin-first-architecture` | **Date**: 2026-02-06 | **Spec**: `specs/004-plugin-first-architecture/spec.md`
**Input**: Feature specification from `specs/004-plugin-first-architecture/spec.md`

## Summary
Migrate the SDD framework from a monolithic structure (33 skills, 29 agents, 3 hooks, 43 scripts in flat directories) to a composable plugin-first architecture using Claude Code's native plugin system. This enables hot-swap capability modules, per-plugin RL feedback, multi-agent swarm orchestration with cost controls, and marketplace distribution. The migration is phased across 5 releases (v3.2.0 → v4.1.0) to maintain backwards compatibility.

## Technical Context
**Language/Version**: Bash (hooks, scripts), JavaScript/CJS (governance hook), Markdown (agents, skills, commands), JSON (manifests)
**Primary Dependencies**: Claude Code CLI (plugin system, --agents, --max-budget-usd, --fallback-model, --permission-mode)
**Storage**: Filesystem (plugin directories, state files, plugin.json manifests)
**Testing**: Shell script validation, hook integration tests, plugin install/enable/disable tests
**Target Platform**: macOS/Linux (Claude Code CLI environments)
**Project Type**: Single (framework toolkit — not a web/mobile app)
**Performance Goals**: Plugin enable/disable <2s, agent spawn <5s, token overhead <10%
**Constraints**: Must maintain constitutional governance across all plugins, backwards compatible during transition
**Scale/Scope**: 13 plugins, 28 skills, 15 agents, 12 commands, 3 hook event types

## Phase 0: Outline & Research

Research complete — see `research.md`. Key decisions:

7. ✅ Coexistence mode for backwards compatibility

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| 13 plugins (Principle V) | Granular selective installation per domain | Fewer plugins (5) bundles unrelated domains, losing key benefit |
| tmux for swarm coordination | Event-driven agent notification | Polling wastes resources; no native IPC in Claude Code |
| Coexistence mode during transition | Backwards compatibility for downstream projects | Big-bang migration would break all downstream projects |
